  \begin{itemize} 
\item \textbf{Pure term in impure methods.} In project \codefont{Apache accumulo}, a pure term \codefont{get} was used in an impure method \codefont{getEntries} that mutates its parameter \codefont{tablets} in an \codefont{if} condition. If this condition fails, this \codefont{getEntries} method will throw an exception to its caller, forcing the program to sleep for 3 seconds. This bug was fixed in revision \href{https://github.com/apache/accumulo/commit/446a37a#diff-002fc1274cd98bf226ee8398551261a6}{446a37} with a refactoring. 
\item \textbf{Impure term in a pure method.} As shown in Part (A) on the right, in project \codefont{Apache hadoop}, an impure term \codefont{prepare} was used in a pure method \codefont{prepare} which simply returns \codefont{true}.  When developer tried to add a new feature for \codefont{HADOOP-489}, she encountered a \codefont{NullPointerException} for the \codefont{prepare} method. The developer fixed this bug in revision \href{https://github.com/apache/hadoop/commit/7c2d77#diff-c0169961c9fbca7c0a7b3afb2049665a}{7c2d77}  by implementing another \codefont{init} method and invokes this new method in \codefont{prepare} method. This fix changes  the method \codefont{prepare} to impure method.
% In project \codefont{Apache hadoop}, an impure term \codefont{start} was used in a pure method \codefont{startSkipping}. The method \codefont{startSkipping} checks if the bad records can be skipped and returns the checking result. This method experiences  a bug fix which sets a maximum boundary for skipped records in revision \codefont{d18d34}.
\item \textbf{Inconsistent object names in similar type use patterns.} In project \codefont{Apache hbase}, there was a field named \codefont{flushListener} with type \codefont{FlushRequester}, while other object names with similar type use patterns used \codefont{*Requester} as a name. This inconsistent name was introduced in revision \href{https://github.com/apache/hbase/commit/e1da900df#diff-b9d8ad451ab3fd5f5ff8593268230822}{e1da90} when user renamed the class \codefont{CacheFlushListener} to \codefont{FlushRequester} yet forgot to rename the object name consistently. This inconsistent name was renamed as \codefont{flushRequester} in fix revision \href{https://github.com/apache/hbase/commit/f52130953245d93489a9c700c3b00eedbe70fdb9/src/main/java/org/apache/hadoop/hbase/regionserver/HRegion.java}{f52130}. 
 \end{itemize}
